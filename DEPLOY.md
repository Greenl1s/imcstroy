# Развёртывание на своём сервере

## Что где будет работать

```
             ИНТЕРНЕТ / ЛОКАЛЬНАЯ СЕТЬ
                       │
                    :80 :443
                       ▼
        ┌──────────────────────────────┐
        │  Caddy                       │  сертификат HTTPS,
        │  отдаёт сайт, проксирует /api│  статика приложения
        └───────────┬──────────────────┘
                    │  внутренняя сеть Docker
        ┌───────────▼──────────────────┐
        │  API на Node                 │  проверяет пароль и роль,
        │  порт 3000, наружу закрыт    │  только он ходит в базу
        └───────────┬──────────────────┘
                    │
        ┌───────────▼──────────────────┐
        │  PostgreSQL                  │  порт наружу НЕ проброшен
        │  данные в docker-томе        │
        └──────────────────────────────┘
```

Достучаться до базы из интернета нельзя в принципе — у неё нет открытого порта.
Всё, что видно снаружи, — сайт и `/api`.

## Что нужно

* Любая машина с Linux: VPS (от 1 ГБ памяти достаточно) или компьютер в офисе.
* Docker и Docker Compose.
* Домен — если приложение должно работать из интернета. Для работы только
  внутри локальной сети домен не нужен.

Установка Docker на Ubuntu/Debian:

```bash
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER   # перелогиньтесь после этой команды
```

## Запуск

```bash
# 1. Скопировать проект на сервер и перейти в его папку
cd pribory

# 2. Заполнить настройки
cp .env.example .env
nano .env
```

Что вписать в `.env`:

| Переменная | Чем заполнить |
|---|---|
| `POSTGRES_PASSWORD` | Длинный пароль. Вручную его вводить не придётся |
| `JWT_SECRET` | Результат команды `openssl rand -base64 48` |
| `ADMIN_PASSWORD` | Пароль первого администратора, минимум 8 символов |
| `SITE_ADDRESS` | Домен (`pribory.example.ru`) или `:80` для локальной сети |

```bash
# 3. Запустить
docker compose up -d --build

# 4. Посмотреть, что всё поднялось
docker compose logs -f api
```

Готово. Открывайте адрес сервера в браузере и входите под администратором,
которого задали в `.env`.

**Сразу после первого входа смените пароль администратора** через кнопку «Профиль».

### Домен

Если используете домен: заведите A-запись, указывающую на IP сервера, и откройте
порты 80 и 443. Caddy сам получит сертификат Let's Encrypt и будет его продлевать —
делать для этого ничего не нужно.

### Без домена (только локальная сеть)

Оставьте `SITE_ADDRESS=:80`. Приложение будет доступно по адресу вида
`http://192.168.1.50`. Это допустимо внутри своей сети, но пароли при этом ходят
без шифрования — в интернет такое выставлять нельзя.

## Перенос данных из Supabase

Один раз, после первого запуска:

```bash
docker compose exec api sh -c '
  SUPABASE_URL=https://ваш-проект.supabase.co \
  SUPABASE_KEY=ваш-anon-ключ \
  node ../scripts/import-from-supabase.js'
```

Скрипт перенесёт пользователей, приборы (включая списанные), фотографии и историю.
Пароли будут захэшированы, входить можно будет с прежними.

> **Важно.** Старые пароли лежали в базе открытым текстом и были доступны любому,
> кто открывал приложение. Считайте их скомпрометированными: после переноса
> попросите всех сменить пароль.

Старые номера приборов сохранятся как инвентарные — переклеивать наклейки
и перепечатывать QR-коды не придётся.

## Резервные копии

Все данные лежат в одной базе, поэтому копия — это один файл.

```bash
# Создать копию
docker compose exec -T db pg_dump -U pribory pribory | gzip > backup-$(date +%F).sql.gz

# Восстановить
gunzip -c backup-2026-07-12.sql.gz | docker compose exec -T db psql -U pribory pribory
```

Ежедневная копия в 3 часа ночи — добавьте в `crontab -e`:

```
0 3 * * * cd /path/to/pribory && docker compose exec -T db pg_dump -U pribory pribory | gzip > /var/backups/pribory-$(date +\%F).sql.gz
```

Проверьте хотя бы раз, что копия действительно восстанавливается. Резервная копия,
которую никогда не пробовали развернуть, — это не резервная копия.

## Обновление приложения

```bash
git pull
docker compose up -d --build
```

Данные при этом не теряются: они лежат в томе `db-data`, отдельно от контейнеров.

## Полезные команды

```bash
docker compose ps                  # что запущено
docker compose logs -f api         # логи сервера
docker compose restart api         # перезапустить API
docker compose exec db psql -U pribory pribory   # консоль базы
docker compose down                # остановить (данные сохраняются)
```

## Если что-то не работает

**«Нет связи с сервером» в браузере** — проверьте `docker compose logs api`.
Чаще всего это незаполненный `JWT_SECRET` (нужно минимум 32 символа) или
пустой `ADMIN_PASSWORD` при первом запуске.

**Сертификат не выпускается** — Caddy не может подтвердить владение доменом.
Проверьте, что A-запись домена указывает на этот сервер и что порт 80 открыт
(он нужен для проверки, даже если сайт работает по 443).

**Порт 80 занят** — на сервере уже есть nginx или Apache. Либо остановите их,
либо проксируйте `/` и `/api` на Caddy с другого порта.
